# tcpdump と OSI 7層

`01-network/README.md` の手順で `sudo tcpdump -A -i lo0 'port 3000'` を使ったが、
そもそも tcpdump が「どの層で」「どうやって」通信を捕まえているのかを補足する。

---

## tcpdump とは

ネットワークカード (NIC) を流れる **生のパケットをコピー** して、人間に
見える形に整形するツール。

中身は 2 つに分かれる:

- **libpcap** — パケットを OS カーネルから取り出すライブラリ
- **tcpdump** — それを使って表示する CLI

Wireshark も中身は同じ libpcap。GUI 版だと思えばよい。

---

## OSI 7層モデル

| 層 | 名前 | 例 | 単位 |
|---|---|---|---|
| 7 | アプリケーション | HTTP, SMTP, DNS | メッセージ |
| 6 | プレゼンテーション | TLS の暗号化/復号 | — |
| 5 | セッション | セッション管理 | — |
| 4 | トランスポート | TCP, UDP | セグメント |
| 3 | ネットワーク | IP, ICMP | パケット |
| 2 | データリンク | Ethernet, Wi-Fi | フレーム |
| 1 | 物理 | 電気信号、電波 | ビット |

実務では TCP/IP 4 層モデルで語ることが多いが、説明上は 7 層が分かりやすい。

## tcpdump がいる場所

**捕まえるのは Layer 2 (フレーム単位)、表示は Layer 2〜7 を解釈して出す。**

```
   ┌──────────────────────────┐
   │  Layer 7: HTTP の文字     │ ← 平文なら文字として表示できる
   │  Layer 6: TLS 暗号化      │ ← ここで暗号化されると、上は読めない
   │  Layer 4: TCP ヘッダ      │ ← ポート番号、シーケンス番号など
   │  Layer 3: IP ヘッダ       │ ← 送信元/宛先 IP
   │  Layer 2: Ethernet フレーム │ ← ★ ここでカーネルからコピーを貰う
   │  Layer 1: 電気信号        │
   └──────────────────────────┘
```

「**Layer 2 で盗み聞きして、Layer 7 まで解釈して見せる**」のが tcpdump。

## どうやって捕まえているか

### 通常の流れ

```
   電波/電気
     ↓
   NIC (ネットワークカード) … 自分宛のフレームだけ拾って上に渡す
     ↓
   OS カーネル
     ↓
   普通のアプリ (ブラウザ等)
```

### tcpdump が居るとき

```
   電波/電気
     ↓
   NIC … (必要なら "プロミスキャスモード" で全フレーム拾う)
     ↓
   OS カーネル ── BPF/AF_PACKET でコピーを横取り ──→ libpcap → tcpdump
     ↓
   普通のアプリ
```

ポイント:

- **盗む** のではなく、**カーネルが用意した正規の覗き窓 (BPF)** を使う。
  だから `sudo` (管理者権限) が要る。
- 通常モードでは「自分宛 + ブロードキャスト」しか見えない。
- **プロミスキャスモード** にすると「自分宛じゃないフレーム」も拾える。
  ただし今どきはスイッチや Wi-Fi の暗号化で実際には届かないことも多い。

---

## HTTPS が読めない理由 (OSI 視点)

| 通信 | tcpdump で見えるもの |
|---|---|
| HTTP | Layer 7 (HTTP テキスト) まで丸見え。`password=p@ss` も読める |
| HTTPS | Layer 3 (IP) と Layer 4 (TCP) は見える。**Layer 6 で暗号化**されているので Layer 7 は乱数のような bytes になる |

HTTPS は「**経路上には居られても、Layer 6 から上が読めない**」状態を作っている。
これが TLS の効能を OSI で言い直したもの。

## コマンドの分解 (再掲)

```
sudo tcpdump -A -i lo0 'port 3000'
       │      │  │ │      └── フィルタ式 (条件)
       │      │  │ └────── 監視するインターフェース
       │      │  └──────── interface 指定オプション
       │      └─────────── 中身を ASCII で表示
       └──────────────── raw packet を読むので管理者権限
```

### interface (`-i`)

| 名前 | 何の通信 |
|---|---|
| `lo0` | 自 PC 内のループバック (localhost ↔ localhost) |
| `en0` | 有線 / Wi-Fi など実際のネットワーク |
| `any` | 全インターフェース |

### フィルタ式

省略すれば全部見える。書き方の例:

```bash
tcpdump -i en0 'host 192.168.1.10'        # 特定ホスト
tcpdump -i en0 'port 443'                  # 特定ポート
tcpdump -i en0 'host x.com and port 443'   # 組み合わせ
tcpdump -i en0                             # フィルタなし = 全部
```

---

## まとめ

- tcpdump = libpcap を使った、Layer 2 のパケットコピー機
- カーネルの BPF 経由で覗くので `sudo` が必要
- 表示は Layer 2〜7 を全部デコードしてくれる
- ただし暗号化されている層から上は中身が読めない (これが TLS の意味)
- 攻撃者目線では「経路上に居られるか」が成立条件。HTTPS にすると、居られても拾えなくなる
